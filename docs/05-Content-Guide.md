# 05 - Content Guide

Australian English. Friendly, confident, expert. Explain technical
topics clearly. Build trust before selling.

Build for conversions first, design second.
Every page must have a clear CTA above the fold.
Every service page must match one Google Ads intent.
Never send paid traffic to a generic page.
Do not mix ecommerce products into service landing pages.
Use internal links naturally.
Use unique SEO metadata on every route.
Use JSON-LD schema where relevant.
Keep mobile performance high.
Use reusable components and data-driven page templates.
Use ASAP brand colours only.
Keep tone confident, friendly and local.

Service-page content rules:

- Give each service unique symptoms, overview, process, features, FAQs and related links.
- Explain technical terms in plain Australian English and distinguish testing from assumptions.
- Do not invent prices, completion times or guaranteed outcomes.
- State that customers bring vehicles to the Ringwood workshop; never imply mobile service.
- For caravan work, state that capability depends on system type, access and repair scope.
